final关键字详解

发表于 2019-01-14 | 更新于 2022-04-17 | 分类于 Java | 评论数： | 阅读次数：
阅读时长 ≈ 2 分钟

一、final基本概念

  • 被 final 修饰的类不可以被继承（因此 final 不能修饰抽象类）

  • 被 final修饰的方法不可以被重写

  • 被 final 修饰的变量不可以被改变，这里有两种情况

    • 如果被修饰的变量是基本类型，则值是不能做更改的
    • 如果被修饰变量是引用类型，则不能更改引用地址，但是可用更改被引用的内部内容

二、final的作用

  • 使用 final 修饰的类或变量，可以清楚的避免因为意外赋值导致编程错误
  • 使用 final 修饰的只读数据，可以在并发情况下，减少额外的同步开销，以及一些防御性的拷贝必要

三、题外话

  • 如何使得对象不可变？我们知道 final 修饰的对象只是引用地址不可变，而引用的内容不是固定的。那么如何使对象不可变
    1. 将 class 自身声明为 fnal ，这样别人就不能扩展来绕过限制了
    2. 将所有成员变量定义为 private 和 fnal ，并且不要实现 setter 方法
    3. 通常构造对象时，成员变量使用深度拷贝来初始化，而不是直接赋值，这是一种防御措施，因为你无法确定输入对象不被其他人修改
    4. 如果确实需要实现 getter 方法，或者其他可能会返回内部状态的方法，使用 copy-on-write 原则，创建私有的对象，避免别人操作对象改变原有的属性
  • final 在什么情况下赋值操作
    • 类加载过程中，在准备阶段，final 修饰的类的静态属性直接赋值操作，而不是默认值（基本类型 0 、引用类型为 null ）
    • 类实例化过程中，final 变量初始化程序定义的值
